###### 进程间通信的几种方式及其特点
- 目的
    1. 数据传输：一个进程将数据传送给另一个进程。
    2. 共享数据：多个进程要操作的数据。
    3. 通知事件：一个进程通知另一个进程某个事件已经发生。
    4. 资源共享：多个进程共享同样的资源。
    5. 进程控制：一个进程希望控制另一些线程的执行（如 debug 线程）

- 方式
    1. 管道
    2. 信号量
    3. 共享内存
    4. 信号
    5. 消息队列
    6. socket 套接字

- 特点
    1. 管道
        - 半双工，数据单向流动。
        - 数据只能在有亲缘关系的进程之间通信。
        - 缺点是速度慢，数据容量小，能在父子进程间使用。
    2. 信号量
        - 不能传递复杂消息，容量小，用来同步解决进程安全问题。
    3. 共享内存
        - 容易控制容量，速度快，但要保证同步，类似于线程安全。
    4. 信号
        - 主要作为进程间以及线程之间的同步手段。
    5. 消息队列
        - 容易受系统限制，要注意第一次读的时候，要考虑上一次数据没有读完的问题。
